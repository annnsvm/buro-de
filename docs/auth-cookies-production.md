# Налаштування cookies у продакшені

Інструкція для викату бекенду з cookie-based авторизацією (access_token, refresh_token).

## 1. Змінні середовища

У продакшені в `.env` (або в змінних середовища на хостингу) задай:

| Змінна | Опис | Приклад для продакшену |
|--------|------|------------------------|
| **COOKIE_SECURE** | `true` — cookies тільки по HTTPS | `true` |
| **CORS_ORIGIN** | Дозволений origin фронту (один або кілька через кому) | `https://app.buero.de` або `https://buero.de,https://www.buero.de` |
| **COOKIE_DOMAIN** | Опційно. Якщо фронт і API на різних піддоменах — спільний домен (з крапкою на початку), щоб cookie була доступна обом | `.buero.de` або порожньо, якщо один домен |

Інші змінні (JWT_*, DATABASE_URL тощо) налаштовуються як зараз.

## 2. HTTPS

У продакшені API має працювати **тільки по HTTPS**. Інакше при `COOKIE_SECURE=true` браузер не збере cookie.

Переконайся, що:
- домен має валідний SSL (наприклад Let's Encrypt);
- редирект HTTP → HTTPS увімкнений (на рівні балансера/проксі або сервера).

## 3. CORS і credentials

У коді вже є `credentials: true`. Для продакшену важливо:

- **CORS_ORIGIN** має **точно** збігатися з origin фронту: схема + хост + порт (наприклад `https://app.buero.de` без слеша в кінці).
- Якщо фронтів кілька — перелічи їх через кому: `https://app.buero.de,https://admin.buero.de`.
- Не використовуй `*` для origin, якщо відправляєш cookies — браузер це заблокує.

## 4. Cookie Domain (якщо API і фронт на різних піддоменах)

Приклад:
- API: `https://api.buero.de`
- Фронт: `https://app.buero.de`

Щоб cookie, встановлені з `api.buero.de`, відправлялись на запити з `app.buero.de`:

1. У продакшені задай `COOKIE_DOMAIN=.buero.de` (крапка перед доменом = для всіх піддоменів).
2. CookieService вже підтримує цю змінну: якщо вона задана, cookies встановлюються з атрибутом `domain`.

Якщо API і фронт на одному домені (наприклад `buero.de` з шляхами `/api` і `/`) — **COOKIE_DOMAIN** можна не ставити.

## 5. SameSite

У коді використовується `sameSite: 'lax'`. Для продакшену це зазвичай достатньо: cookie відправляються при top-level навігації та при запитах з того ж сайту. Якщо потрібна інтеграція з зовнішніми редиректами (OAuth тощо), можна розглянути `sameSite: 'none'` (обов'язково з `secure: true`).

## 6. Чеклист перед викатом

- [ ] HTTPS увімкнений для API та фронту.
- [ ] `COOKIE_SECURE=true` у продакшен .env.
- [ ] `CORS_ORIGIN` = точний URL фронту (або список через кому).
- [ ] Якщо API і фронт на різних піддоменах — налаштовано `COOKIE_DOMAIN`.
- [ ] На фронті всі запити до API з `credentials: 'include'` або `withCredentials: true`.
- [ ] Перевірено в браузері: після логіну з'являються `access_token` і `refresh_token` з path `/api`, і вони відправляються при запиті на `/api/users/me`.
